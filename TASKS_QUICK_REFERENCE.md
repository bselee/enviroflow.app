## Meeting Agendas

### Weekly Standup (15 min)
1. **Completed:** Last week's tasks + % complete
2. **Doing:** Current week assignments
3. **Blockers:** Issues preventing progress
4. **Risks:** Timeline/quality concerns

### Phase Gate Review (1 hr)
1. **Demo:** Show completed work
2. **Code review:** Pass/fail on quality criteria
3. **Feedback:** Issues to fix before Phase N+1
4. **Sign-off:** Ready for next phase?

## Checklists

### Before Starting Task

- [ ] Read full task description in ENVIROFLOW_PROJECT_TODOS.md
- [ ] Review acceptance criteria (are they testable?)
- [ ] Check dependencies (are blockers unblocked?)
- [ ] Understand design (sketches, mockups available?)
- [ ] Setup branch: `feat/task-[ID]-[brief-name]`

### Before Submitting PR

- [ ] Code compiles (npm run build works)
- [ ] Linting passes (npm run lint)
- [ ] Tests pass (npm run test)
- [ ] TypeScript strict mode passes
- [ ] All acceptance criteria met
- [ ] PR description links to task
- [ ] Screenshots/video for UI changes
- [ ] Tested on mobile (if applicable)

### Before Phase Review

- [ ] All tasks in phase merged to main
- [ ] No outstanding review feedback
- [ ] Performance validated
- [ ] Security checklist passed
- [ ] Documentation updated
- [ ] Team confident in quality
